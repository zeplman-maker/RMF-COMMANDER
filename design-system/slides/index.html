01 · THE PROBLEMFOUR GAPS DRIVING THE NEED

Operators have no real-time
portfolio surface.

An ISSM managing 26 systems and supporting more than one million end users cannot answer a real-time AO question without returning to a workstation.
GAP 01

No mobile visibility

ATO status and POA&M counts require a full eMASS desktop session. ISSMs can't answer real-time AO questions from the field.
GAP 02

Documentation burden

Control statements, POA&M entries, and monitoring summaries written manually each cycle. 40–60% of ISSM time on documentation.
GAP 03

Reactive awareness

Critical CVEs and STIG updates discovered during scheduled scans — not in real time. Findings can sit open for days before the ISSM sees them.
GAP 04

ATO expiration risk

18 systems tracked by manual calendar reminders. One missed renewal forces emergency action or IATT extensions.

02 · PROPOSED SOLUTIONRMF COMMANDER — CORE CAPABILITIES

A command surface that
assists, never decides.

Portfolio dashboard

ATO status, expiration countdown, POA&M severity counts, and control posture. Mobile + web. 15-minute refresh cycle.

Smart alert engine

90/60/30-day ATO alerts. POA&M milestone slips. eMASS workflow state changes. Push to enrolled devices.

AI documentation assist

Draft control statements and POA&M language grounded in NIST SP 800-53 Rev. 5. Human approval required before any use.

Natural language query

Ask portfolio questions in plain English. Answers grounded in authorized eMASS data only. Source cited every time.

Executive briefing gen

One-tap AO and PM posture summaries. PDF and PPTX output. Validated against live data at generation time.

Zero device persistence

No eMASS data written to device storage, cache, or clipboard. Session wipe on background. Derived-credential auth only.

06 · RISK & CONTROLSRISK REGISTER AND MITIGATIONS

Every identified risk has
a structural mitigation.

Risk Impact Mitigation
AI output treated as official approval High Visible AI-DRAFTED label, human approval gate, no auto-submission, mandatory user training.
Unauthorized access to eMASS data High Role-scoped access, BFF architecture, derived credentials, full audit logs with 1-year retention.
Sensitive data exposed via notifications Med Payload limited to system name and alert type. Full detail requires authenticated session.
Production integration approved too early High Pilot limited to mock/test/sandbox. Production requires separate AO approval — explicit gate.
Model hallucination Med RAG grounding, source citations on every claim, uncertainty flags, mandatory human review.
