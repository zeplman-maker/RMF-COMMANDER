1. Purpose
This concept paper requests support for a controlled RMF COMMANDER prototype and 90-day pilot. The pilot uses mock, sandbox, or test data only. It does not access production eMASS data, process classified information, submit artifacts, or make authorization decisions.
2. Operational Need
ISSMs and ISSOs lack purpose-built mobile tooling for RMF portfolio visibility.
Authorization status, POA&amp;M counts, control posture, and scan freshness currently require desktop eMASS review.
Routine control statements, POA&amp;M language, and monitoring summaries consume an estimated 40 to 60 percent of ISSM time.
ATO expiration windows, POA&amp;M milestone slips, CVE activity, and STIG updates are often tracked manually or reactively.
An ISSM managing 26 systems and supporting more than one million end users needs faster portfolio visibility without changing AO, SCA, ISSM, or ISSO authority.
3. Proposed Solution
RMF COMMANDER is a controlled mobile and web platform that gives authorized RMF personnel secure visibility into portfolio status, ATO timelines, POA&amp;M risk, vulnerability awareness, and documentation support. The AI layer assists. It does not decide.
Capability
What It Provides
Pilot Boundary
Portfolio Dashboard
ATO status, days to expiration, POA&amp;M counts by severity, control implementation posture, and scan freshness.
Read-only test, sandbox, or mock data.
Smart Alerts
90/60/30-day ATO alerts, POA&amp;M milestone slip alerts, and eMASS workflow state change notifications.
Notification payload limited to system name and alert type.
AI Documentation Assistant
Draft control statements, POA&amp;M language, monitoring summaries, and briefing language grounded in approved references.
Draft only. Visible AI-drafted label. Human approval required before use.
Natural Language Query
Plain-language questions answered using authorized RMF data only.
No cross-system inference outside role scope.
Executive Briefing Generator
Portfolio posture summaries formatted for AO, PM, and leadership updates.
Validated against the current test dataset during pilot.
4. AI Governance
NIST AI RMF Function
RMF COMMANDER Control
GOVERN
Human-in-the-loop mandate, role-scoped access, audit logs, and no AI authority to make compliance decisions.
MAP
Use case limited to assistive drafting and query. No authorization decisions, no SCA replacement, no AO replacement.
MEASURE
Pilot metrics include AI draft acceptance, source citation accuracy, alert accuracy, user confidence, and gate adherence.
MANAGE
Flagged AI outputs reviewed within 24 hours. AO receives 30-day touchpoint briefings. Scope changes require separate approval.
5. Approval Requested
Approve controlled prototype development and a 90-day pilot using mock, sandbox, or test data only.
Authorize stakeholder validation, architecture review, and test data coordination.
Endorse eMASS REST API access request for test/sandbox provisioning only.
Approve Phase 1 activities: DoD cloud tenant planning, MDM development-device enrollment, and eMASS system registration for RMF COMMANDER itself.
Establish recurring 30-day AO touchpoints throughout the pilot.
