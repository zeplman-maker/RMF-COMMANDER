1. Request
Request test and sandbox eMASS REST API access to support the RMF COMMANDER controlled prototype and 90-day pilot. The request is limited to non-production data and does not authorize production data access, write-back, artifact submission, or operational use.
2. Justification
RMF COMMANDER requires API-style data access during the pilot to validate the feasibility of a secure portfolio dashboard, ATO expiration alerting, mock POA&amp;M posture reporting, source-cited natural language query, and future eMASS-integrated workflows. Test/sandbox access allows the team to validate architecture and controls before any production integration decision.

4. Security Controls
Client application never communicates directly with eMASS.
DoD-authorized cloud backend mediates all service calls.
API credentials remain in Azure Key Vault or AWS Secrets Manager.
Backend-to-eMASS communication uses mutual TLS.
User sessions use derived PIV credentials on MDM-enrolled devices.
No eMASS package content persists on the client device.
All API calls are logged and available for review.
5. Approval Limitation
This request is limited to test/sandbox provisioning. Production eMASS integration is a separate future decision that requires additional AO approval and completion of the RMF COMMANDER authorization pathway.
